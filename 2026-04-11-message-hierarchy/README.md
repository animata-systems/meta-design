# Иерархия сообщений — 2026-04-11

Сессия посвящена философии иерархии сообщений в Corazon Protocol. Отправная точка — неудовлетворённость плоской моделью `server/channel/post/comment` и поиск подхода, который даёт гибкость без потери дисциплины. Результат — рамка из трёх ортогональных измерений (scope, structure, discipline), ряд принятых архитектурных решений и унифицированная модель, в которой канал — единственная сущность scope, а «сервер» — частный случай канала.

## Главные выводы одной выжимкой

**Три измерения: scope / structure / discipline.** Все три — проекции одного append-only лога транзакций. Scope — кто здесь (membership). Structure — что из чего выросло (граф сообщений). Discipline — что здесь можно (правила канала).

**Нода — валидатор, не хранитель.** Невалидные транзакции отклоняются и не попадают в лог. Чистый лог → чистый снапшот → чистый рейтинг.

**Discipline: конечный набор из пяти правил, enforce на ноде.** Discipline сужает права, данные scope, но никогда не расширяет. Добавление нового правила — upgrade протокола.

**Канал — единственная сущность scope.** Два типа: `conversation` (в нём общаются) и `space` (в нём группируют другие каналы). Дерево через `parent_scope`. «Сервер» — не отдельная сущность, а space верхнего уровня.

**Адресация — плоские id в протоколе.** `channel-id/msg-id`. Навигация по дереву scope — задача приложения. Терминология протокола и терминология UI/UX — разные вещи.

**Триада Corazon: Stake / Rank / Settlement.** Три балансирующие силы сети. Stake — стоимость присутствия (rate limit, vouch, penalty). Rank — стоимость доверия (SymbioRank, четыре оси, decay). Settlement — стоимость ценности (экономический каркас, рыночные цены, x402). Три замкнутых контура обратной связи: убери любую силу — две оставшихся коллапсируют. Scope/structure/discipline — триада каналов. Stake/Rank/Settlement — триада сети. Две триады, один протокол.

**Rank — одновременно фасад и несущая стена.** Защитный механизм (sybil-защита через многомерность) и первый продуктовый опыт (четыре характеристики, видимые с первой транзакции). Базовый SymbioRank работает с первого дня параллельно с базовой нодой.

## Связь с предыдущими сессиями

Сессия развивает `2026-04-08-message-bus`, где принято решение о едином `message` и дереве через `parent`. Здесь тот же паттерн (одна сущность, дерево через parent-ссылку) применён на уровне каналов.

Учитывает выводы `2026-04-10-corazon-protocol`: транзакционный фундамент (всё — проекция лога), нода-валидатор, Corazon как открытая сеть.

Позиция по «серверу» — развитие дискуссии из `2026-04-10`, где обсуждалось, является ли server_id полем протокола или понятием приложения. Здесь найден третий путь: server — не отдельное поле и не чистый app-level, а канал типа space с `parent_scope`.

## Статус

**Рабочий снимок понимания на 2026-04-11.** Ничего из этого не является нормой. Рамка scope/structure/discipline ощущается устойчивой. Конкретный набор правил discipline — черновой, требует проверки на большем количестве сценариев. Ceiling (наследование discipline от parent space) — принят с оговоркой.
